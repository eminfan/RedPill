# 人类模因图谱 · 数字人引擎构建 v1.0

> 本框架将一个人拆解为**可运行的模因引擎配置**——不是分析报告，而是能在未知场景下预测此人行为的数字人模拟器。
> 核心设计目标：**即使执行者没有心理学背景，严格按流程走也能构建出有预测力的数字人模型。**
> 实现方式：所有深度机制都编码为**硬性规则**而非建议。不满足自检条件则滑梯退出。

---

## 执行总则（硬性）

1. **事实优先**：所有模因分类必须先锚定可验证的行为/言论证据。不能基于感觉或印象做分类。
2. **置信度强制**：每个模因分类必须附带置信度。置信度由证据质量自动决定（见M-Map置信度规则）。
3. **对称性强制**：DIM-01~05行为规则中，任何"激活条件"必须同时写"抑制条件"，任何"共生关系"必须同时写"拮抗关系"。只写一半=字段未完成=自检不通过。
4. **反事实强制**：引擎构建完成后，必须经过"攻击自己结论"的步骤（见Phase 3回测+反事实测试）。未完成反事实测试=引擎标记为未验证。
5. **推测标记**：所有超出事实范围的推断必须显式标注（`[推断]`/`[推测]`/`[推测·敏感]`），未标注=自检不通过。
6. **自检关卡不可跳过**：每个阶段末尾的自检关卡必须逐项打勾通过。任何一项不通过→滑梯退出并输出原因。

## 阶段零：入口分流

### 0.1 触发识别

收到用户输入后，先判断是否匹配以下触发模式：

| 模式 | 示例 | 路径 |
|------|------|------|
| 明确人名 + 触发词 | 「模因蒸馏乔布斯」「构建数字人马斯克」「模因分析毛泽东」 | **直接路径** |
| 明确人名 + 意图 | 「XX的模因图谱」「XX的底层逻辑」「XX是什么样的人」 | **直接路径** |
| 模糊需求 | 「我想理解XX这个人的行为逻辑」「帮我分析XX的思维方式」 | **直接路径**（提取人名） |
| 不明确 | 「分析一个人的行为模式」 | **澄清路径** → 追问具体人名 |

**直接路径**确认后，提取目标人名 → 进入 0.2。

### 0.2 需求澄清

```
目标人物：______
确认：______（用户确认正确）
聚焦方向：______（全面画像 / 聚焦特定领域，默认全面）
本地语料：______（用户是否有此人的一手素材？书籍/访谈/视频/文章等）
```

如果用户提供了本地语料 → 标记为**本地语料模式**，Phase 1 优先分析本地素材，网络搜索为补充。
如果用户只说「就做XX」没有更多信息 → 默认全面画像 + 无本地语料（走网络搜索），直接推进。

### 0.3 创建输出目录

收到确认后立即执行，在调研之前完成：

```

├── references/                  ← 调研过程文件
│   ├── 01-公开言论.md
│   ├── 02-决策记录.md
│   ├── 03-行为模式.md
│   ├── 04-外部视角.md
│   └── 05-时间线.md
└── 数字人SKILL.md               ← 生成的 `.claude/skills/[person]-meme-sim/SKILL.md` 副本

### ⏏ 自检关卡0

```
□ 目标人物已确认
□ 输出目录已创建
□ 聚焦方向已明确（全面/聚焦）
□ 本地语料模式已确认（是/否）
□ 如果是中国人物：信息源策略已标记（排除知乎/微信公众号/百度百科，优先B站原始视频/小宇宙播客/权威媒体）
```

**未通过则滑梯退出**：输出「入口条件不满足——本分析因【具体原因】终止」，结束。

## 阶段一：多源信息采集

启动 5 个并行 Agent，每个负责不同信息维度。

### 5 个 Agent 的任务分配

| Agent | 采集目标 | 最低数量 | 输出文件 |
|-------|---------|---------|---------|
| **A1 公开言论** | 此人公开发表的观点、立场、价值观声明（书籍/演讲/访谈/社交媒体） | ≥5 条 | `01-公开言论.md` |
| **A2 决策记录** | 此人做出的重大决策（职业/商业/人生/政治），含背景+选择+结果 | ≥3 个 | `02-决策记录.md` |
| **A3 行为模式** | 此人反复出现的行为特征、处事风格、人际互动模式 | ≥3 个模式 | `03-行为模式.md` |
| **A4 外部视角** | 他人对此人的评价、批评、传记资料，含正面和负面 | ≥2 个来源 | `04-外部视角.md` |
| **A5 时间线** | 关键人生节点、转折事件、思想/立场变化 | ≥5 个节点 | `05-时间线.md` |

### 每个 Agent 的硬性要求

- 调研结果必须写入 `references/0X-xxx.md`
- 区分三个层级：
  - **事实**：「他说了X」「他做了Y」——有可查来源
  - **模式归纳**「[推断]：他在Z场景下反复表现出W行为」——基于≥2个事实的逻辑归纳
  - **动机推测**「[推测]：他做X是为了Y」——无直接证据的分析者解读
  - **可疑归因**「[推测·敏感]：他有意X是为了服务于Y的隐性目的」——动机归因
- 发现矛盾记录矛盾，不要调和（矛盾本身是有价值的信号）
- 至少采集1条「此人做了什么但说了相反的话」的言行不一致案例（如存在）

### 本地语料模式

如果用户提供了本地素材，执行顺序：
1. 先分析本地素材，按 5 个 Agent 维度分类
2. 识别信息缺口（哪些维度不足）
3. 只对缺口维度启动网络搜索补充
4. 来源标记：明确区分「本地素材」vs「网络搜索」

### Agent 超时与失败处理

- **单个 Agent 超时**：不等待，继续推进。在 Phase 2 中标注「信息不足」并降低该维度置信度
- **信息源匮乏**（全部 Agent 合计 < 10 条可用来源）：Phase 0 就提醒用户降低期望，Phase 2 所有判断强制标注`[推测]`
- **Agent 结果冲突**：保留矛盾，在 M-Map 的「模因冲突」字段收录

### ⏏ 自检关卡1

```
□ A1 公开言论 ≥ 5 条已收录
□ A2 决策记录 ≥ 3 个已收录
□ A3 行为模式 ≥ 3 个已收录
□ A4 外部视角 ≥ 2 个来源已收录
□ A5 时间线 ≥ 5 个节点已收录
□ 所有采集内容已区分事实/推断/推测层级（推测标记已使用）
□ 如存在矛盾信息，已在文件中保留
□ 信息源标注已完成（本地/网络/训练数据）
```

**任何一项不通过→滑梯退出**。输出「信息不足——本分析因【具体原因】不达标而终止」，结束。

## 阶段二：模因引擎构建（CORE）

### 2.1 M-Map：静态骨架

将 5 个 Agent 采集的素材，按 M1-M6 分类体系逐一映射。

**M-Map 置信度规则**：

| 置信度 | 条件 |
|-------|------|
| 高 | 该模因类型有 ≥ 2 个独立事实来源支撑，且无矛盾行为 |
| 中 | 有 1 个直接事实来源，或 ≥ 2 个推断[推断]支撑 |
| 低 | 仅基于推测[推测]，或无直接证据的归纳 |
| 未确定 | 信息不足，无法判定 |

### ⏏ 自检关卡3

```

## 迭代修正协议

如用户对引擎配置有异议，或后续获得新信息：

1. **定位偏差来源**——M-Map 分类错了？DIM 规则不对？场合映射不准？还是信息不足？
2. **补充信息**——回到 Phase 1 对应 Agent 补充采集
3. **修正引擎**——从 Phase 2 受影响部分开始重做
4. **重新验证**——重新执行 Phase 3 回测和反事实测试
5. **输出修正版**——标明修改部分和原因，更新数字人 SKILL.md
