# 文学作品模因图谱分析 Prompt v3 —— 通用增强版

> 版本：v3.0
> 定位：在 v2 完整框架基础上，补充「分析前检索校准」「第一刀通用切入」「多轮迭代流程」三大缺失环节，解决跨作品泛化能力短板
> 核心改进：不再依赖分析者对作品的先验认知，通过检索驱动 + 结构化第一刀 + 迭代校准，确保任何作品都能产出深度不低于鹿鼎记案例的分析

---

## 一、使用总纲：三阶段工作流

本提示词不再是一次性提交的指令，而是**三阶段迭代工作流**：

```
阶段一：检索校准（分析前）
  → 大模型先完成事实锚定检索 + 作品类型识别 + 认知偏差标注
  → 人工确认检索结果是否完整，再进入阶段二

阶段二：深度拆解（核心分析）
  → 基于校准后的事实锚定，执行完整六层级分析
  → 产出完整模因图谱

阶段三：闭环验证（分析后）
  → 大模型自行验证分析结论是否闭环
  → 人工反馈偏差 → 大模型嵌入修正 → 定稿
```

---

## 二、阶段一：分析前检索校准指令（前置锚定）

**核心目的**：在执行任何分析之前，先对作品对应的真实事实做系统检索与校准，确保后续所有分析建立在可靠的事实基础上。这是 v2 中最缺失的环节。

### 2.1 作品类型自动识别指令

提交分析请求时，大模型必须先执行作品类型识别：

```
【强制步骤：作品类型识别】
在开始任何分析之前，先根据以下分类标准，识别目标作品的核心类型：

A. 历史题材——作品涉及真实历史人物/事件/朝代，有可对照的正史记载
B. 现实主义题材——作品以当代/近代现实社会为背景，涉及真实存在的社会结构/行业/群体
C. 纯虚构世界观——作品设定在完全虚构的世界，不直接对应真实历史或现实
D. 改编作品——有明确原著/原始素材的改编
E. 功能文本——说明书、教程、规章、宣传材料等以传递信息而非叙事为目的的文本
F. 混合型——兼有以上多种类型特征（如三体：科幻虚构 + 历史映射 + 现实批判）

识别结果决定后续检索策略和分析侧重点。
```

### 2.2 分类型检索清单（强制前置检索）

根据作品类型，执行对应的检索指令，检索结果写入"事实锚定底表"。

#### A类 · 历史题材检索指令

```
【历史题材·强制前置检索】

请从以下维度检索目标作品对应的真实历史事实，建立事实锚定底表：

1. 核心历史事件：
   - 作品涉及的每个关键事件，对应真实历史上发生了什么？
   - 事件的时间、地点、参与方、结果是什么？
   - 主流史学界对此事件的历史定性是什么？

2. 核心历史人物：
   - 作品中出现的真实历史人物，其真实生平是什么？
   - 该人物在正史中的评价是什么？（关注正史而非民间传说）
   - 该人物的完整履历中，作品刻意呈现了哪些、回避了哪些？

3. 时代背景的宏观事实：
   - 该朝代/时期的经济总量、人口数据、科技水平、文化成就是什么？
   - 同时期世界其他文明的对比情况如何？
   - 该时期的民生水平、社会结构、制度特征是什么？

4. 作品后设史（元历史）：
   - 这部作品本身是在什么历史背景下创作的？
   - 作者所处的时代与作品描写时代之间，发生了什么历史变迁？
   - 作品创作时，其目标受众的主流历史认知是什么？

检索完成后输出「事实底表」格式：

## 事实锚定底表
| 维度 | 作品叙事 | 真实事实 | 偏差类型 |
|------|---------|---------|---------|
| 事件1 | 小说说... | 历史上是... | 美化/抹黑/回避 |
| 人物1 | 小说说... | 历史上是... | 美化/抹黑/回避 |
| ... | ... | ... | ... |
```

#### B类 · 现实主义题材检索指令

```
【现实主义题材·强制前置检索】

请从以下维度检索目标作品对应的现实事实：

1. 作品涉及的社会问题：
   - 作品描写的核心社会矛盾（贫富分化、阶层固化、灰色经济等），在中国的真实数据是什么？
   - 基尼系数、阶层流动率、行业集中度等可量化指标
   - 相关领域的主流学术研究结论是什么？

2. 作品设定的行业/领域：
   - 作品描写的行业（建筑、餐饮、互联网、制造业等）的真实运作逻辑是什么？
   - 该行业的利益分配结构、权力层级、上下游关系
   - 该行业公认的核心规则与潜规则

3. 作品的创作背景：
   - 作者的真实身份、行业背景、公开立场
   - 作品创作时官方对该领域的舆论导向是什么？
   - 主流评论界对这部作品的评价集中在什么方向？

4. 受众的社会结构位置：
   - 这部作品的主要受众是哪个社会营养级的群体？
   - 受众对该作品的核心情感共鸣是什么？
   - 这种共鸣是来自亲身经历，还是来自传媒构建的认知？

检索完成后输出「现实锚定底表」。
```

#### C类 · 纯虚构世界观检索指令

```
【纯虚构世界观·强制前置检索】

对于完全虚构世界观作品，事实锚定转向「框架锚定」：

1. 世界观的内在逻辑：
   - 作品世界的基本运作规则是什么？（魔法体系/科技设定/社会制度）
   - 这些规则是否存在内在矛盾或不一致？
   - 规则的设计服务于什么叙事目的？

2. 现实隐喻识别：
   - 虚构世界的权力结构和现实中哪种结构相似？
   - 虚构世界的阶级/种族/物种关系，映射了现实中的什么关系？
   - 作者是否有公开言论暗示作品的现实指向？

3. 作品的文化谱系：
   - 这部作品继承了哪些已有作品的叙事传统/模因？
   - 它与同类虚构世界相比，在底层立场上有什么差异？
   - 它的创新之处是纯粹的技术创新，还是立场/叙事的创新？

4. 事实锚定的替代方案：
   - 不是对照"真实事实"，而是对照"读者的默认认知"
   - 作品在刻意挑战读者的哪些默认认知？
   - 这种挑战是服务于唤醒还是服务于驯化？
```

#### F类 · 混合型作品特殊指令

```
【混合型作品·额外检索】

除主类型的检索指令外，额外执行：
1. 识别作品中"有事实锚点"的部分和"纯虚构"的部分
2. 对每个有事实锚点的元素单独检索
3. 特别注意：混合型作品往往用虚构部分掩护事实部分的叙事立场
```

#### E类 · 功能文本特殊指令

```
【功能文本·强制前置检索】

功能文本的分析逻辑与叙事作品完全不同：

1. 功能定位：
   - 这份文本的核心功能是什么？（指导操作/传递规章/说服购买/建立认知）
   - 它的目标受众是谁？目标受众在使用前的认知状态是什么？
   - 它想让受众在阅读后做什么？不做什么？

2. 利益链条：
   - 从这份文本中受益最大的群体是谁？
   - 它的传播维护了什么结构？削弱了什么结构？
   - 如果受众完全按文本指导行动，谁得利、谁受损？

3. 信息完整性：
   - 文本披露了所有必要信息，还是选择性披露？
   - 文本省略了什么受众应该知道的信息？
   - 信息不对称的受益方是谁？

4. 认知框架：
   - 文本预设了读者已经认同什么？拒绝什么？
   - 它把什么问题定义为了"需要解决的问题"？
   - 它把什么解决方案定义为"唯一合理的方案"？
```

### 2.3 认知偏差自动标注

检索完成后，大模型必须自动标注自己的认知局限性：

```
认知偏差标注（强制输出）：
1. 我检索到的信息来自哪些信源？是否存在信源偏向？
2. 我对作品对应的领域具备充分知识吗？哪些维度的信息可能不完整？
3. 作品类型识别是否准确？是否存在交叉类型的可能？
4. 需要分析者补充/确认的关键事实有哪些？
```

### 2.4 人工确认节点

```
【阶段一完成 · 人工确认节点】

事实锚定底表已输出。请确认：
1. 核心事实是否准确？
2. 是否有遗漏的重要维度？
3. 认知偏差标注是否接受？

确认后进入阶段二分析。
```

## 三、阶段二：核心分析指令

### 3.1 第一刀通用切入模板

**核心目的**：不依赖对作品的先验认知，通过标准化切入路径找到分析突破口。这段补充在六层级第一层级中，作为"事实锚定完成后的第一刀"。

```
【第一刀通用切入：谁受益？谁受损？】

事实锚定完成后，执行以下标准化切入步骤：

第一步：标记受益者和受损者
在作品中找到以下两组角色/群体：
A组（受益者）：作品中地位提升、获得成功、被正面呈现的角色/群体
B组（受损者）：作品中地位下降、遭受失败、被负面呈现的角色/群体

第二步：追问偏差
对于A组：他们的成功在真实世界中发生的概率有多大？作品是否掩盖了他们成功的结构性条件（资源、关系、运气、特权）？
对于B组：他们的失败在真实世界中是必然的吗？作品是否掩盖了导致他们失败的系统性因素？

第三步：识别叙事矛盾
A组和B组的设定之间是否存在逻辑矛盾？
  例如：作品一边说"努力就能成功"，一边把高启强写成"好人被迫变坏"
  这种矛盾指向了作品的底层真实根模因

第四步：定位利益服务方向
综合以上三步，回答：
- 这部作品的叙事最终让谁受益？
- 这部作品的叙事让谁受损？
- 这个受益-受损结构，是符合事实的，还是偏离事实的？

这个四步切入法适用于任何作品类型，不依赖分析者的先验认知。
```

### 3.2 完整的六层级分析流程

使用 v2 版本已有的六层级分析流程骨架，但将第六层级重构为面向普通用户的格式：

1. **第一层级：事实锚定与立场底层**（事实锚定已在阶段一完成，此阶段直接使用结果）
2. **第二层级：根模因层**
3. **第三层级：传播模因复合体**
4. **第四层级：受众驯化效果**
5. **第五层级：模因互作网络**
6. **第六层级：终极判定**（重构版，见下方 3.3）

**唯一补充**：每一层级分析完成后，增加一句对本层级分析的**置信度评估**：

```
置信度标记：
- [高置信度]：事实锚定充分，逻辑链条完整，结论可靠
- [中等置信度]：部分维度信息可能不完整，结论需谨慎
- [低置信度]：事实锚定不充分，此部分结论可能偏差，建议人工复核
```

### 3.3 第六层级：终极判定（面向普通用户的格式）

**核心变化**：不再使用"红丸""天道""营养级"等内部术语，而是用直白的结论回答三个问题。内部术语放在括号里作为注释，不干扰可读性。

```
【终极判定】

问题一：这部作品站在哪一边？
→ [明确结论] 它是在帮人看清真相，还是在帮人保持麻木？
   （内部判定：共生性天道启蒙 / 掠夺性天道驯化）
   一句话理由：____________________

问题二：它的传播服务于谁？
→ [明确结论] 谁从这部作品的广泛传播中受益？谁受损？
   （内部判定：服务对象在哪个社会层级，维护/解构了什么结构）
   一句话理由：____________________

问题三：如果你看穿了它，该怎么做？
→ [可操作建议] 作为读者，看完分析后你实际可以做什么：
   - 防御：如何在接触这部作品时保持清醒
   - 利用：如何从中提取有价值的部分
   - 传播：是否值得向他人推荐，以及推荐时需要注意什么

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【可选 · 理论推导说明】（仅对想了解理论背景的读者展示）

以下内容使用内部框架，供延伸阅读：

a) 天道属性定位：
   - 共生性天道：打破掠夺结构，促进认知觉醒
   - 掠夺性天道：维护掠夺结构，固化认知
   - 中性：不涉及立场

b) 社会营养级定位：
   - 服务生产层：让底层安心生产，提供上升通道幻觉
   - 服务传递层：维持信息传递，固化认知框架
   - 服务富集层：维护掠夺结构，消解反抗意识

c) 驯化/启蒙强度判定：
   - 强驯化 → 弱驯化 → 中性 → 弱启蒙 → 强启蒙

判定依据：事实处理、人物塑造、台词效果、反抗意识、利益服务五个维度的综合评分。
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 四、阶段三：闭环验证与迭代校准流程

### 4.1 第一轮自校验（大模型自行执行）

分析完成后，大模型自动执行闭环验证：

```
【闭环自校验】

请回答以下三个问题，验证分析结论是否闭环：

Q1：这套分析能无矛盾地解释「作品为什么要刻意扭曲/美化对应事实」吗？
如果回答不了，说明事实锚定不够深，需要回到阶段一补充检索。

Q2：这套分析能解释「作品对受众产生了什么可验证的长期认知影响」吗？
如果回答不了，说明第四层级的分析不够具体，需要补充受众影响的现实证据。

Q3：这套分析能回答「作品的传播最终服务于谁的利益」吗？
如果回答不了，说明底层立场的分析没有穿透，需要回到第二层级重新识别根模因。

三轮验证通过后，标记为「PASS」。
未能通过的维度，标注具体缺口。
```

### 4.2 人工反馈校准模板

第一轮分析产出后，向分析者提供校准入口：

```
【人工校准入口】

如果以下任一情况出现，请指出：
1. 分析结论和你的认知存在明显偏差？
2. 分析遗漏了你认为关键的信息？
3. 分析中的判断前提本身有问题？

请用以下格式反馈偏差：

```
反馈编号：[1/2/3...]
偏差位置：[哪个层级的哪个结论]
偏差描述：[你认为正确的方向是什么]
补充信息：[你掌握但大模型不知道的]
```

收到反馈后，执行修正迭代。
```

### 4.3 修正与迭代协议

大模型收到人工反馈后的处理流程：

```
【修正协议——收到反馈后的执行步骤】

当分析者提供偏差反馈后，不要立即修改结论，按以下步骤执行：

Step 1：理解偏差
- 先分析偏差来源：是事实锚定错了？逻辑链条错了？还是信息缺失？
- 输出偏差的根因定位

Step 2：补充检索
- 基于偏差反馈，回到阶段一补充检索缺失的信息维度
- 更新事实锚定底表

Step 3：重新推导
- 基于更新后的事实底表，从第一层级开始重新推导受影响的相关层级
- 不要只改被指出的那一段——因为事实锚定变了，下游分析可能都要变

Step 4：输出修正版
- 在修正版中标明：哪些部分修改了，修改前的版本是什么，修改的原因是什么

Step 5：二次验证
- 重新执行闭环自校验
- 确认修正没有引入新的逻辑矛盾
```

### 4.4 迭代终止条件

```
【迭代终止判断】

满足以下任一条件时终止迭代：
1. 分析者确认"PASS"（无进一步偏差）
2. 连续两轮反馈指向同一维度但每次修正方向相反（说明该维度信息不足，标记为"不确定性"保留而非强行定论）
3. 修正轮次达到上限（最多3轮迭代，第3轮后无论是否完全满意，标记剩余争议点后定稿）
```

## 五、完整工作流示例（以《狂飙》为例）

### 5.1 阶段一执行示范

假设分析者提交《狂飙》分析请求，大模型执行：

```
【作品类型识别】
识别结果：B类 · 现实主义题材（以2000-2021年中国社会为背景）

【分类型检索开始·现实事实锚定】

维度1：社会问题数据
- 检索中国2000-2020年基尼系数变化趋势
- 检索中国城乡收入差距、阶层流动率数据
- 检索房地产/建筑行业在该时期的利益分配结构
- 检索中国底层群体（下岗工人、城中村居民）在该时期的生存状况数据

维度2：作品背景检索
- 检索《狂飙》创作团队、编剧立场、采访记录
- 检索该剧的播出背景（2023年扫黑除恶常态化阶段）
- 检索主流评论界对该剧的评价分布

维度3：受众画像
```

### 5.2 阶段二执行示范

```
【第一刀切入：谁受益？谁受损？】

A组（受益者）：
- 高启强：从鱼贩变成黑社会头目 → 现实生活中底层鱼贩变成黑老大的概率极低，作品掩盖了这种跃迁的结构性条件
- 赵立冬：长期受贿未被查处 → 现实中保护伞的暴露概率在2006年后的打黑行动中显著增加
- 旧厂街施工队：跟着高启强有活干 → 现实中这部分人的真实去向是什么？

B组（受损者）：
- 安欣：二十年追凶，满头白发 → 现实中刑警的处境是否真如此绝望？
- 谭思言：举报后被灭口 → 举报者的真实处境
- 底层受害者：被高启强势力压榨的人

【偏差识别】
作品一方面呈现高启强的"魅力"（让观众恨不起来），一方面呈现安欣的"悲剧"（让观众心疼）。
这两种叙事方向同时存在，形成了底层矛盾：作品到底想让人同情高启强，还是想让人同情安欣？

这种矛盾指向了作品的底层策略——用高启强的个人命运展示系统的掠夺性，用安欣的个人牺牲展示系统的正义性。两种叙事服务于不同的受众需求，让不同立场的观众都能找到自己的代入点。
```

### 5.3 阶段三执行示范

```
【闭环自校验】

Q1：能解释作品为什么刻意平衡高启强和安欣的叙事权重吗？
→ 可以。因为作品的传播策略需要同时满足"揭露黑暗"（审查通过的前提）和"人物魅力"（吸引观众的前提）。

Q2：能解释作品对受众的真实认知影响吗？
→ 可以。受众分成两派：一派看到"系统腐败"（觉醒方向），一派看到"人性复杂"（道德模糊方向）。两种解读方向同时存在。

Q3：能解释作品最终服务于谁的利益吗？
→ 中等置信度。官方层面：服务于扫黑除恶的政策宣传需要。受众层面：服务于对"系统不公"的情绪释放（替代性满足）。

三轮自校验：Q1 PASS、Q2 PASS、Q3 中等置信度。
建议人工复核Q3结论——这部作品的传播长期看是强化了对制度的信任还是怀疑？这个判断需要更多现实数据。
```
