# 作品立场分析 · Stance Analyzer

> 输入作品名，自动完成完整立场分析（v6.1框架）。
> 分析一部作品在暗中传递什么价值观、为谁的利益说话、对读者是叫醒还是哄睡。

## 一句话

拆解作品在"暗中传递什么价值观、为谁的利益服务"——是让人更清醒（启蒙），还是让人更麻木（驯化）。

## 安装

```bash
npx skills add stance-analyzer
```

或在 Claude Code 中直接触发：输入「作品立场」「分析作品」「分析这本书」「分析这部电影」「XX的立场」「stance XX」「XX在传递什么」等关键词即可。

## 用法

在 Claude Code 中，输入作品名即可。例如：

- 「分析《活着》」
- 「红楼梦的立场」
- 「stance 三体」
- 「鹿鼎记在传递什么」

分析框架会自动执行完整流程（5层分析+4维交叉验证+反事实测试+自鉴声明），无需用户了解框架细节。

## 框架概要

**四阶段执行**：
1. **检索校准** — 作品类型识别 + 事实锚定底表 + 推测性结论标记
2. **作品分析（五层）** — 素材层→结构层→功能层→调控层→演化层
3. **四维交叉验证** — 文本+传播+社会+效果证据
4. **反事实测试 + 最终判断** — 攻击自己的结论 + 标签+自鉴声明

详见 [SKILL.md](SKILL.md)（可执行分析指令）和 [框架（当前版本）.md](框架（当前版本）.md)（版本索引）。

## 核心设计理念

**用流程保障深度**。框架将深度分析所需的"批判性思维"编码为硬性规则：
- 对称性强制：激活条件必须配抑制条件，共生必须配拮抗
- 拮抗标注：每条素材必须标注与至少一条其他素材的矛盾/支撑关系
- 反事实测试：分析必须三次攻击自己的结论
- 推测标记：动机归因必须显式标注`[推测·敏感]`

即使执行者没有批判性思维天赋，严格按流程走也能产出深度分析。
