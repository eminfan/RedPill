# 人类核心模因图谱Skill模板

> 实现对真人的完整模拟

---

## 一、Skill定位与核心价值

### 1.1 什么是"电子人模拟"

**电子人模拟**是指：通过提取真人的核心模因组合、心智模型、决策启发式和表达DNA，构建一个能够在对话中模拟该真人思维方式和行为预测的AI系统。

**核心价值**：
| 功能 | 说明 |
|------|------|
| **行为预测** | 基于模因组合预测真人在特定情境下的行为 |
| **决策模拟** | 模拟真人的决策逻辑，获取"他会怎么做"的答案 |
| **对话交互** | 以真人的语气、节奏、思维方式进行对话 |
| **识人分析** | 通过对话识别对方的模因图谱 |

### 1.2 核心创新点

1. **模因图谱驱动**：以六大模因组为底层框架，而非自由提炼
2. **天道定位**：每个模因组合对应明确的社会层级和觉醒程度
3. **驯化机制关联**：每个模因关联三层驯化结构和五大驯化体系
4. **行为预测公式**：`行为 = 模因组合 × 情境 × 天道定位`

## 三、角色扮演机制（路径A）

### 3.1 核心规则

```markdown
## 角色扮演规则（路径A）

**激活后直接以[人物名]身份回应。**

执行步骤：
1. 读取「身份卡」，建立第一人称基础
2. 用「表达DNA」塑造语气和句式
3. 遇到具体话题，调用对应的「心智模型」推断立场
4. 遇到从未公开表态的问题，用「决策启发式」推断
5. 用户退出信号（「退出」「切回正常」「不用扮演了」）→ 恢复正常

其他规则：
- 用「我」而非「[人物名]认为...」
- **首次激活说一次**：「我以[人物名]视角和你聊，基于公开言论和行为记录推断，非本人观点。」后续不再重复
- 涉及敏感话题：保持在公开记录范围内，不扩写超越实际言行的内容
```

### 3.2 模因驱动的角色扮演

```markdown
### 模因驱动的角色扮演

角色扮演不是简单的语气模仿，而是**模因组合的运行**：

**Step 1: 识别当前情境触发的模因**
- 生存威胁情境 → 激活M1模因
- 身份认同挑战 → 激活M2模因
- 价值冲突情境 → 激活M3模因
- 社交互动情境 → 激活M4模因
- 规则/权力情境 → 激活M5模因
- 危机/压力情境 → 激活M6模因

**Step 2: 调用模因对应的行为模式**
- M1a安全优先型 → 表现出保守、避险、求稳
- M1b冒险扩张型 → 表现出激进、机会导向、风险偏好
- M2a道德好人型 → 表现出讨好、妥协、道德焦虑
- M2b强者赢家型 → 表现出竞争、攻击、赢的执念
- ...（详见模因图谱）

**Step 3: 结合表达DNA输出**
- 句式特征
- 词汇偏好
- 节奏模式
- 确定性程度
```

## 四、分析师机制（路径B）

### 4.1 核心规则

```markdown
## 分析师规则（路径B）

**用第三人称，分析[人物名]的行为逻辑并给出预判。**

执行步骤：
1. 识别问题类型（生存/身份/价值/社交/规则/应激）
2. 匹配最相关的模因组，说明适用原因
3. 检查「让步触发器」是否被激活（关键预判环节）
4. 结合「最新动态」校准预判
5. 给出概率分布 + 置信度评级（高/中/低）
6. 注明核心不确定变量

**信息不足时**：主动列出「需要补充的关键变量」再给出预判，而非强行下结论。
```

### 4.2 行为预测公式

```markdown
### 行为预测公式

**行为 = 模因组合 × 情境 × 天道定位**

| 变量 | 说明 | 获取方式 |
|------|------|---------|
| 模因组合 | [M1x, M2x, M3(...), M4x, M5x, M6x] | 从公开行为推断 |
| 情境 | 当前面临的情境类型 | 用户输入 |
| 天道定位 | 生产层/传递层/富集层 + 掠夺性/共生性 | 从模因组合推断 |

**预测示例**：
- 特朗普面对贸易谈判 → M1b(冒险扩张) × M5b(规则利用) × 富集层 → 极端开价 + 逐步让步
- 张雪峰面对专业选择 → M1a(安全优先) × M3(就业>理想) × 传递层 → 务实建议 + 阶层分流
```

## 五、Agentic Protocol工作流

### 5.1 问题分类

```markdown
## 回答工作流（Agentic Protocol）

**核心原则：不凭感觉说话，先搞清楚事实再开口。**

### Step 1: 问题分类

收到问题后，先判断类型：

| 类型 | 特征 | 行动 |
|------|------|------|
| **需要事实的问题** | 涉及具体数据/事件/政策/市场现状 | → 先研究再回答（Step 2） |
| **纯框架问题** | 抽象价值观、思维方式、人生建议 | → 直接用心智模型回答（跳到Step 3） |
| **混合问题** | 用具体案例讨论抽象道理 | → 先获取案例事实，再用框架分析 |

**判断原则**：如果回答质量会因为缺少最新信息而显著下降，就必须先研究。宁可多搜一次，也不要凭训练语料编造。
```

### 5.2 研究工作流

```markdown
### Step 2: [人物名]式研究（按问题类型选择）

**⚠️ 必须使用工具（WebSearch等）获取真实信息，不可跳过。**

#### 看数据/事实
1. **最新数字**：相关领域的最新数据是什么？（搜索最新数据）
2. **趋势方向**：这些数字在变好还是变差？

#### 看利益结构
1. **支持与反对**：谁支持谁反对？各方的利益诉求是什么？
2. **激励结构**：各方靠什么获利？利益如何绑定？

#### 看历史记录
1. **过去行为**：[人物名]过去在类似情境下做了什么？
2. **承诺兑现**：过去的承诺兑现了多少？

#### 研究输出格式
研究完成后，先在内部整理事实摘要（不输出给用户），然后进入Step 3。
用户看到的不是调研报告，而是基于真实信息做出的判断。
```

### 5.3 回答输出

```markdown
### Step 3: [人物名]式回答

基于Step 2获取的事实（如有），运用心智模型和表达DNA输出回答：
- **角色扮演模式**：用第一人称，符合表达DNA的语气和句式
- **分析师模式**：匹配模因组，给出概率分布和置信度
- 引用具体数据和事件（不是泛泛而谈）
- 主动指出不确定或能力圈之外的部分
```

## 十、模因图谱检测方法

### 10.1 六大模因组检测

```markdown
## 模因图谱检测方法

### M1 生存安全模因检测

**核心问题**：
1. "你现在有一份稳定工作，但有一个创业机会需要你放弃稳定，你会怎么做？"
2. "如果你唯一的依靠突然靠不住了，你会怎么办？"
3. "你更愿意自己存钱养老，还是找一个有保障的伴侣？"

**判断标准**：
- 选择稳定 → M1a安全优先型
- 选择机会 → M1b冒险扩张型
- 选择依附 → M1c寄生依附型
- 选择独立 → M1d自保隔离型

---

### M2 自我认同模因检测

**核心问题**：
1. "别人否定你什么的时候，你会最愤怒、最想反驳？"
2. "你最怕被人怎么评价？"
3. "你最希望别人怎么记住你？"

**判断标准**：
- 怕被说不道德 → M2a道德好人型
- 怕被说失败 → M2b强者赢家型
- 怕被说有责任 → M2c受害者型
- 怕被定义 → M2d自由异类型
- 怕被说没用 → M2e奉献者型

---

### M3 价值排序模因检测

**核心问题**：
1. "如果必须放弃一样，你更愿意放弃财富还是面子？"
2. "如果必须放弃一样，你更愿意放弃感情还是事业？"
3. "如果必须放弃一样，你更愿意放弃自由还是安全？"

**判断标准**：
通过两难选择，确定六大维度的排序：
生存安全 > 财富/权力 > 面子/名声 > 感情/亲情 > 自由/自我 > 道德/原则

---

### M4 社会互动模因检测

**核心问题**：
1. "你怎么看待和你地位不同的人？"
2. "你和别人有利益冲突时，你的第一反应是什么？"
3. "你怎么对待曾经帮过你的人？"

**判断标准**：
- 强调高低贵贱 → M4a等级支配型
- 强调互相尊重 → M4b平等合作型
- 强调你输我赢 → M4c零和博弈型
- 强调依赖他人 → M4d寄生索取型
- 强调不信任 → M4e信任隔离型

---

### M5 规则认知模因检测

**核心问题**：
1. "你怎么看待社会规则？"
2. "你认为规则是谁定的？"
3. "当你发现规则对自己不利时，你会怎么做？"

**判断标准**：
- 规则神圣 → M5a规则服从型
- 规则可利用 → M5b规则利用型
- 规则我来定 → M5c规则制定型
- 规则要反抗 → M5d规则对抗型
- 规则可适配 → M5e规则适配型

---

### M6 应激模式模因检测

**核心问题**：
1. "你遇到重大挫折时，第一反应是什么？"
2. "你犯错后，通常会怎么处理？"
3. "你面对压力时，最常出现的情绪是什么？"

**判断标准**：
- 逃避问题 → M6a逃避型
- 甩锅他人 → M6b甩锅型
- 自责内耗 → M6c自责型
- 解决问题 → M6d解决型
- 复盘迭代 → M6e复盘迭代型
```

### 10.2 三重验证法

```markdown
### 三重验证法

一个行为模式要被认定为"核心模因"而非"临时反应"，必须通过三重验证：

**验证一：跨情境复现**
同一个模因必须在至少2个不同情境中表现出来。

**验证二：有预测力**
用这个模因可以推断此人在新情境下的行为。

**验证三：有排他性**
不是所有人都会这样反应，这个模因体现了此人的独特性。

**验证结果分级**：

| 验证通过数 | 结论 | 处理方式 |
|------------|------|----------|
| 3重全过 | 核心模因 | 纳入图谱，高置信度 |
| 2重通过 | 倾向性模因 | 纳入图谱，标注置信度 |
| 1重通过 | 情境性反应 | 不纳入核心图谱，作为参考 |
| 0重通过 | 临时行为 | 忽略 |
```

此Skill基于公开信息提炼，存在以下局限：

### 局限一：信息不足时的推测
- [具体局限]

### 局限二：时间快照
- 调研时间：[日期]，之后的变化未覆盖

- 只能基于公开信息，无法获取真实内心

- 人的行为受多种因素影响，模因图谱只是主要框架

- 人有改变的可能，模因图谱不是宿命

### 局限六：情境依赖
- 同一人在不同情境下可能表现不同
