# 文学作品模因图谱分析框架 v6.1

> **当前版本：v6.1** | 上一版本：v5.8 | 下一版本规划中
> 
> 本文件是框架的索引和变更记录。完整的可执行分析指令见 [SKILL.md](SKILL.md)。
> 
> **设计定位**：供 Claude Code 等大模型直接执行的立场分析框架。
> **产出**：五层分析（L1素材层→L2结构层→L3功能层→L4调控层→L5演化层）+ 四维交叉验证 + 反事实测试 + 终极判定

---

## 与 v5.8 的核心区别

v6.1 是一次**机制升级**而非结构调整。五层结构（L1-L5）和四阶段流程没有大改，但在每个阶段内嵌入了**防止执行者偷懒的硬性约束**。

### 设计原则（贯穿所有版本）

1. **事实优先** — 所有分析先锚定真实事实
2. **置信度强制** — 每个判断必须附带置信度
3. **对称性强制** — 激活条件必须配抑制条件，共生必须配拮抗
4. **反事实强制** — 分析必须经过"攻击自己"的步骤
5. **自检关卡不可跳过** — 不满足条件则滑梯退出

## 快速使用

在 Claude Code 中输入作品名即可触发自动分析：

- 「分析作品《活着》」
- 「立场分析 鹿鼎记」
- 「stance 三体」
- 「还珠格格在传递什么」

详细用法见 [SKILL.md](SKILL.md)。
