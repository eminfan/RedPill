# 模因多组学：跨维度证据交叉验证标准

> 版本：2.0
> 定位：在 3.0 五层产品结构之上，定义一套完整的模因多组学分析范式
> 核心问题：模因分析的所有结论都来自分析者一个人的文本判断——只有一个"组"的证据，结论就没有可靠性
> 核心解决：将基因组学的多组学工作流映射到模因分析，用四个独立证据维度的交叉验证取代单源判断
> 前置阅读：本文是《文学作品模因图谱 3.0 —— 基于人类基因组图谱类比的结构设计》的配套标准

---

## 一、从"一个组"到"多个组"

### 1.1 问题的本质

当前模因图谱所有 L 层的结论（偏差标注、染色体分组、功能注释、演化判定）都来自同一个源头——分析者的文本阅读。

这相当于基因组学只做全基因组测序（WGS），然后通过序列本身去猜每个区域的功能、调控、演化关系。没有 RNA-seq、没有 ChIP-seq、没有敲除实验——只有一个"组"。这在基因组学里是不可接受的。

模因分析要走出"分析者个人判断"的信誉困境，必须引入独立于文本分析的多个证据维度，让结论建立在多个独立来源的交叉验证之上。

### 1.2 基因组学多组学工作流

```
                     ┌──────────────┐
                     │ 全基因组测序  │
                     │    (WGS)     │——→ 序列
                     └──────┬───────┘
                            │
          ┌─────────────────┼─────────────────┐
          ▼                 ▼                  ▼
   ┌────────────┐   ┌──────────────┐   ┌──────────────┐
   │ 转录组学   │   │   表观组学   │   │   功能组学   │
   │ (RNA-seq)  │   │  (ChIP-seq)  │   │  (敲除实验)  │
   │ 哪些被转录 │   │ 调控元件绑定 │   │ 功能验证    │
   └────────────┘   └──────────────┘   └──────────────┘
                            │
                            ▼
                   ┌────────────────┐
                   │   比较组学     │
                   │ (跨物种比对)   │
                   │ 同源/趋同判定  │
                   └────────────────┘
                            │
                            ▼
                   ┌────────────────┐
                   │   综合判定     │
                   │ 交叉验证后结论 │
                   └────────────────┘
```

**关键特征**：
1. 各"组"是独立实验方法，不是同一推理的不同步骤
2. 各"组"从不同角度回答同一个问题
3. 结论的置信度取决于**有多少个独立"组"指向同一个方向**
4. 如果有"组"的结论与其他"组"冲突，优先采纳有更多独立验证支持的结论

### 1.3 模因多组学的映射设计

基因组学的每个"组"对应模因分析中的一种独立证据类型。每种证据类型有自己独立的采集方法、分析工具和验证逻辑。

```
基因组学                模因多组学               采集方法            独立性保证
────────                ──────────              ──────            ────────
全基因组测序            E1 文本组学             系统阅读+标注      分析者直接读作品
(WGS)                   (Text-Omics)            

RNA-seq                 E2 传播组学             外部数据检索       不依赖文本阅读
(转录组)                (Transmission-Omics)                        

ChIP-seq/ATAC-seq       E3 调控组学             社会结构分析       独立于文本内容
(表观组)                (Regulation-Omics)                         

敲除实验/表型关联       E4 效果组学             受众反馈采集       独立于分析者判断
(功能组)                (Effect-Omics)                             

跨物种比对              跨作品/跨文化比对        — 复用 E2+E4 —    
(比较组)                (Comparative-Omics)                       
```

**关键区别**：这不是 L1-L5 的线性流程的替代品，而是在 L1-L5 的外围增加四个独立的"观测窗口"。L1-L5 描述"作品里有什么"，E1-E4 验证"这个结论有多可靠"。

## 二、四组证据维度

### E1：文本组学（Text-Omics）

**对应基因组学**：全基因组测序（WGS）。

**核心问题**：模因在文本中"写了什么"？

**独立性**：这是基准组——当前模因图谱已有。它的结论是后续所有验证的起点，但不应该作为唯一依据。

**采集方法**：分析者的系统阅读 + 标准化标注。
- L1 层0：原始素材清单（台词/设定/叙事选择/符号）
- L1 层1：偏差标注（美化/抹黑/回避/置换/无偏差）
- 事实锚定底表：作品叙事 vs 事实对照

**产出**：模因序列清单 + 偏差标注集。

**已有工具**：模因图谱模板-空白.md 的 L1 部分；3.0 结构设计文档的 L1 章节。

**不需要额外设计。** 以下三个维度的设计才是新的。

---

### E2：传播组学（Transmission-Omics）

**对应基因组学**：RNA-seq（转录组——哪些基因被转录了）。

**基因组学问题**：基因组序列中有很多区域（~98%）不编码蛋白。RNA-seq 回答"哪些区域实际被转录成了 RNA"，过滤掉沉默的序列。

**模因分析问题**：文本中有大量内容——不是所有内容都成为了"模因"。E2 回答"哪些模因实际进入了传播、被复制和变异了"，过滤掉只在纸面上存在的内容。

**独立性保证**：E2 的数据来自外部可观测的传播事实，不是来自文本阅读。分析者不需要重新读作品，只需要观察作品在现实世界中的传播轨迹。

**数据来源说明**：本节列出的"数据来源"分三种模式——联网模式（有网络可用）、离线手动模式（无网络）、大模型辅助模式（有LLM但无联网）。每种模式产出同一格式的E2证据报告，但采集方法等级不同。

---

#### E2-a：转录检测——哪些模因被复制了

**核心操作**：拿E1层0的素材清单，逐条检查它们在改编版本中是否被保留、删除或改变。

| 数据维度 | 采集内容 | 识别方法 |
|---------|---------|---------|
| 跨媒介复制 | 作品的核心设定和台词在影视/戏剧/游戏/动漫等改编中被保留了多少次？ | 逐版本对照：改编版本 vs 原版，标记保留/删除/改变的设定 |
| 跨文化复制 | 翻译版保留了哪些模因？失落了哪些？ | 不同语言版的对比阅读（或依赖翻译研究文献） |
| 独立引用 | 作品的台词/设定脱离原文后是否被独立引用？引用频率？ | 搜索引擎、社交媒体趋势、语料库 |

**三种采集模式**：

```
联网模式（P1~P2级）：
  数据来源：维基百科、豆瓣条目、影视数据库
  操作：搜索"作品名+改编"，列出所有改编版本，逐版本对比核心设定的保留情况
  优势：数据全、版本完整
  局限：需要网络

离线手动模式（P2~P3级）：
  数据来源：分析者的作品知识库（看过/记得的改编版）+ 可接触到的实体资料
  操作流程：
    Step 1 - 制作改编版本清单：凭回忆列出所有已知改编版本（电视剧/电影/游戏等），
             对每个版本标注"我看过/我知道有/我听说过"
    Step 2 - 核心设定保留检查：取E1层0的素材清单，对每个素材问"这个设定在[版本X]中
             出现了吗？→ 被强化了还是弱化了？→ 被删除了还是被改写了？"
    Step 3 - 标注记忆可信度：对每条检查结果标注[A]确定（我清楚记得）/ [B]较确定
             （有印象但不完全确认）/ [C]推断（我推测如此，需要验证）
    Step 4 - 标记缺口：对没看过的版本，标注为"待补看"，不计入最终判定
  优势：不依赖网络、可完成、每个数据点可追溯
  局限：依赖分析者的记忆覆盖度，未看过的版本只能标记为缺口

大模型辅助模式（P1~P3级）：
  提示词结构：将E1素材清单和已知改编版本列表输入LLM，让LLM逐条分析保留情况
  关键要求：
    - 要求LLM对每条结论标注依据来源（训练数据/推断/猜测）
    - 要求LLM在不确定时标记"不确定"而非强行回答
    - 分析者对LLM输出做交叉检查，修正已知错误
  优势：能快速生成初稿
  局限：LLM可能编造改编版本或设定保留情况，必须交叉检查
```

---

#### E2-b：变异检测——模因在复制中改变了什么

**核心操作**：对E1识别的核心模因（尤其是台词和设定），追踪它们在传播中是否发生了意义偏移。

| 数据维度 | 采集内容 | 识别方法 |
|---------|---------|---------|
| 意义漂变 | 被引用的短语/设定在使用中是否发生了意义偏移？ | 语境采样：收集 20-50 个独立使用实例，标注使用语境 |
| 功能转化 | 模因的功能是否从"驯化"转变为"娱乐"或其他？ | 对比原始语境和当代语境的情感倾向 |
| 二次创作 | 用户的自主改编（同人/戏仿/翻拍）是强化还是解构了原模因？ | 同人平台（AO3/Lofter/B站）的内容分析 |

**三种采集模式**：

```
联网模式（P1~P2级）：
  数据来源：社交媒体API、同人创作平台、B站/短视频平台
  操作：搜索核心台词/设定的当代引用，采集20-50个独立使用实例，逐条标注使用语境
  优势：数据规模大，可定量分析
  局限：需要网络或API权限

离线手动模式（P2~P3级）：
  数据来源：分析者日常接触到的引用实例（记忆采样法）
  操作流程：
    Step 1 - 核心模因清单：从E1中选出3-5个最可能被独立引用的模因（通常是台词），
             如"鸟生鱼汤""反清复明=抢钱抢女人"等
    Step 2 - 记忆采样：凭回忆收集每个模因的独立使用实例。不追求完整，而是标注
             "我在以下语境中见过/听过这个引用……"
    Step 3 - 语境分类：对每个记忆实例标注使用语境——[严肃引用] / [搞笑梗] /
             [反讽用法] / [无意义口头禅] / [其他]
    Step 4 - 漂变轨迹绘制：按时间顺序排列不同语境的使用，观察是否有意义漂变的
             趋势（如"拍马屁称赞→搞笑梗→反讽"）
    Step 5 - 可信度标注：每条记忆标注[A]清晰记忆 / [B]模糊印象 / [C]推测
  注意事项：
    - 分析师自己的日常观察也是数据，关键是诚实地标注"这是我接触到的样本"
    - 不要假装有系统性采样——"我在豆瓣看过至少20条评论提到鸟生鱼汤">"我系统
      采集了20条评论"。后者是P2，前者是P3
    - 如果记忆中的实例太少（<5个），标记为"样本不足"，不做漂变判定

大模型辅助模式（P2~P3级）：
  提示词结构：输入核心模因清单，要求LLM列举它"知道"的引用实例和漂变轨迹
  关键要求：
    - 要求LLM区分"训练数据中学到的常见用法"vs"自己的推断"
    - 要求LLM按使用语境分类并给出每个类别的估计比例
    - 分析者用自己知道的实例做抽样验证
  局限：LLM可能将少数例子夸大为普遍趋势。要求它标注"这个用法是主流还是边缘"
```

#### E2-c：表达量分析——模因在不同人群中的表达强度

**核心操作**：同一个作品，不同人群的接受方式不同。分析模因在不同受众群体中的表达差异。

| 数据维度 | 采集内容 | 识别方法 |
|---------|---------|---------|
| 代际差异 | 不同年龄段的受众对同一作品的核心记忆点是否不同？ | 分年龄段采样（不同平台的用户年龄分布有差异） |
| 地域差异 | 不同文化区域的受众对同一模因的接受度差异 | 跨平台/跨语言数据对比 |
| 社会层级差异 | 不同教育水平/社会背景的受众对模因的解读差异 | 评论来源的上下文推断 |

**三种采集模式**：

```
联网模式（P1~P2级）：
  数据来源：分平台评论采样（豆瓣 vs 知乎 vs B站 vs 微博——用户画像不同）
  操作：在不同平台搜索同一作品的相关讨论，比较讨论焦点的差异
  优势：平台用户画像可作为代际/地域的代理指标
  局限：平台画像需要额外知识支撑

离线手动模式（P3~P4级）：
  数据来源：分析者对受众群体的观察 + 合理推断
  操作流程：
    Step 1 - 受众分层：根据分析者对不同年龄/地域/背景群体的接触经验，列出
             可能存在的解读差异。标注每个分层依据的来源——"我观察到"vs"我推断"
    Step 2 - 差异假设：对每个差异维度提出可检验的假设，如：
             "60-80后读者更关注鹿鼎记的'反武侠'层面，90-00后更关注搞笑层面"
    Step 3 - 假设等级标注：
             [已验证]：我有多个实例支撑这个判断
             [有线索]：我有1-2个实例但不够
             [纯推断]：我没有实例，这是逻辑推测
    Step 4 - 悬置未验证的假设：纯推断的假设不能进入一致性矩阵
  局限：纯推断（P4级）不能作为独立验证
  特别警告：离线模式下的表达量分析最容易混入分析者的预设——"我觉得年轻人应该
  更喜欢搞笑版"可能是真的，也可能只是刻板印象。必须以[已验证/[有线索]/[纯推断]
  明确标记

大模型辅助模式（P3级）：
  提示词结构：输入作品信息和受众分层维度，要求LLM基于训练语料中不同平台的
  讨论特征，给出代际/地域差异的推断
  局限：LLM的训练数据中包含大量网络评论，它"知道"豆瓣用户和B站用户的讨论
  差异。但这是间接知识，不是直接采样。标注为[模型推断]
```

#### E2 离线模式的核心原则

```
原则一：记忆也是数据，但必须标注可信度
  分析者日常接触到的引用实例、看过的改编版本、听过的讨论——这些都是真实数据。
  关键不是"你没有系统采样"，而是"你诚实地标注了数据来源的可靠程度"。

原则二：未覆盖的缺口比虚假的完整更有价值
  离线模式下必然有大量数据缺口。"没看过某部改编版"->标注缺口->对应维度的
  置信度降级。这比"假装知道"然后给出错误结论要好。

原则三：P3/P4级数据只能做"线索"，不能做"验证"
  离线模式下，大部分数据是P3（分析者观察）或P4（理论推断）级。这些数据可以
  用于提出假设、补充说明，但不能作为"独立验证维度"计入一致性矩阵。
  E2只有在至少E2-a达到P2级、E2-b或E2-c至少有一项达到P2级时，才能提供
  有效验证。

原则四：离线E2的结论必须附带采集方式声明
  每条E2结论后必须标注：
  [联网采集] / [离线手动-P2级] / [离线手动-P3级] / [大模型辅助]
```

#### E2 证据产出格式（统一格式，不论采集模式）

```
【E2 传播组学】

作品：《鹿鼎记》
采集模式：[离线手动-P2/P3级] 或 [联网-P1/P2级] 或 [大模型辅助-P3级]

E2-a 转录检测：
  跨媒介复制：
    - 电视剧(4次)：1998TVB陈小春版[我看过-A]——核心模因全部保留，
      韦小宝搞笑化增强，历史背景弱化
    - 电影(2次)：1992周星驰版[我看过-A]——大幅无厘头化，但"反清复明=
      抢钱抢女人"台词保留；2011王晶版[没看过-缺口]
    - 游戏(1次)：2005版[我知道有-B]——弱化了历史背景，保留人物关系
  → E1识别的核心模因在跨媒介传播中被保留——[一致，P2级]

E2-b 变异检测：
  "鸟生鱼汤"漂变轨迹：
    - 原著：拍马屁称赞康熙[我看过-A]
    - 电视剧版：搞笑化演绎[我看过-A]
    - 网络语境：纯搞笑梗，部分反讽[我在B站/微博见过-B]
  → 核心模因从"美化圣君"漂变为"纯娱乐"——[部分变异，P3级]

E2-c 表达量分析：
  代际差异：60-80后更关注"反武侠"[有线索]，90-00后更关注"搞笑"[有线索]
  → E1对模因功能的判定需要按受众群体分情况讨论——[补充，P3级]，未进入
    一致性矩阵判定
```

### E3：调控组学（Regulation-Omics）

**对应基因组学**：ChIP-seq（表观组——什么控制了基因的表达）。

**基因组学问题**：基因序列本身不决定表达。基因是否被转录，受启动子/增强子/沉默子等调控元件的控制。ChIP-seq 回答"哪些区域有蛋白质绑定，可能参与调控"。

**模因分析问题**：模是否存在是一回事，是否被表达是另一回事。社会结构、权力关系、传播渠道——这些因素控制着哪些模因被放大、哪些被抑制。E3 回答"什么社会条件控制了这个模因的表达"。

**独立性保证**：E3 的数据来自社会结构分析，不是来自文本。分析者不讨论"作品想表达什么"，而是讨论"什么社会力量在传播这个作品"。

#### E3-a：权力结构分析——谁在控制模因的传播

| 数据维度 | 采集内容 | 识别方法 |
|---------|---------|---------|
| 体制亲和性 | 作品的底层叙事是否与主流意识形态一致？ | 对照国家话语、教育体系核心理念 |
| 资本路径 | 作品通过什么商业模式传播？谁从中获利？ | 出版/影视/游戏产业链分析 |
| 筛选机制 | 谁在决定"这部作品值得传播"？（出版社/影视公司/平台算法） | 传播链关键节点分析 |

**数据来源**：出版史、媒体经济学、平台研究文献。

#### E3-b：渠道分析——传播渠道如何塑造了模因

| 数据维度 | 采集内容 | 识别方法 |
|---------|---------|---------|
| 传播载体 | 印刷/电视/网络/短视频——不同载体对模因的筛选效果 | 不同载体的内容特征分析 |
| 渠道垄断 | 是否存在单一渠道控制作品传播的情况？ | 市场集中度、平台主导地位 |
| 算法效应 | 推荐算法如何放大了某些模因而抑制了其他？ | 算法特征分析（如YouTube/TikTok的推荐逻辑） |

**数据来源**：传播学文献、平台研究。

#### E3-c：竞争模因分析——什么模因在与它竞争

| 数据维度 | 采集内容 | 识别方法 |
|---------|---------|---------|
| 同时代竞争 | 同一时期有哪些作品/叙事提供了相反的模因？ | 同类作品对照 |
| 抵抗叙事 | 受众是否发展出对抗性解读？ | 批判性评论采样 |
| 替代框架 | 是否有其他理论框架能解释同样的作品特征？ | 替代解释收集（反事实测试） |

**数据来源**：文化研究文献、反事实测试产出。

#### E3 证据产出格式

```
【E3 调控组学】

作品：《鹿鼎记》
采集模式：[分析者独立判断-P2级]

E3-a 权力结构分析：
  体制亲和性：
    - "多民族统一国家"叙事与满清正统叙事高度一致
    - 作品的核心结论（"反抗无理"）与"维稳"话语暗合
    - → 体制层面有强化传播的倾向
  资本路径：
    - 金庸作品是香港文化产业的核心IP
    - 多次影视改编有巨大的商业利益驱动
    - → 商业利益与体制需求叠加，形成双重强化

  同时代竞争：《鹿鼎记》之前的《书剑恩仇录》有较强的汉民族叙事

  替代解释框架：文学性解释（"金庸只是写了个好故事"）——已被反事实测试否决
  → 竞争模因存在但处于弱势——[补充]

### E4：效果组学（Effect-Omics）

**对应基因组学**：敲除实验 + 表型关联（功能组——基因的真实功能是什么）。

**基因组学问题**：序列告诉你"有什么"，转录组告诉你"哪些被转录了"，表观组告诉你"什么控制转录"。但基因到底有什么用？——需要用敲除实验（去掉这个基因看什么表型变化）或人群关联分析（有这个变异的人有什么表型差异）来回答。

**模因分析问题**：文本分析识别了模因（E1），传播分析看到了它的复制（E2），调控分析看到了控制它表达的社会条件（E3）。但这个模因**真的对受众产生了**认知/行为影响吗？——需要看受众的真实反应数据。

**独立性保证**：E4 的数据来自受众的真实反馈，独立于分析者的判断。分析者不推测"应该有什么效果"，而是查找"实际发生了什么效果"。

#### E4-a：受众情感表型

| 数据维度 | 采集内容 | 识别方法 |
|---------|---------|---------|
| 人物情感偏好 | 受众喜欢/讨厌/同情哪些角色？ | 豆瓣短评情感分析、知乎点赞数统计 |
| 价值排序 | 受众在作品设定的两难中选择什么？ | 讨论帖中"站队"分析 |
| 情感传播 | 受众主动向他人传递了哪种情感体验？ | 推荐语情感编码 |

**数据来源**：豆瓣/IMDb 短评、知乎问答、B站评论。

#### E4-b：受众认知表型

| 数据维度 | 采集内容 | 识别方法 |
|---------|---------|---------|
| 事实认知改变 | 受众是否接受了作品扭曲后的事实认知？ | 评论中暴露的历史/现实认知 |
| 框架借用 | 受众是否使用作品的叙事框架来解释现实？ | "用小说逻辑理解现实"的评论 |
| 批判意识 | 是否有受众对作品进行了抵抗式阅读？ | 批判性评论的数量和质量 |

**数据来源**：公开评论、问答平台、"如何看待XXX"类讨论。

#### E4-c：受众行为表型（最深但也最难采集）

| 数据维度 | 采集内容 | 识别方法 |
|---------|---------|---------|
| 消费链效应 | 阅读后是否继续消费同类作品？ | 阅读/观看序列分析 |
| 语言借用 | 日常表达是否引用了作品的语言？ | 社交媒体语言使用分析 |
| 社会行为 | 阅读后是否影响了现实中的价值判断和行为选择？ | 长期追踪研究（超出当前可行范围，标记为"待方法成熟"） |

**数据来源**：前两项——公开数据可分析。第三项——标记为长期研究课题。

**三种采集模式**：

```
联网模式（P1~P2级）：
  E4-a：通过豆瓣/IMDb API或爬虫采集短评，做情感分类（正面/负面/中性），
        统计各个人物的情感倾向比例
  E4-b：通过知乎API或爬虫采集"如何评价XXX"类问题下的回答，做主题建模
        或框架借用编码
  E4-c：消费链效应需要用户行为序列数据（如豆瓣的"看过"列表），部分可采集；
        语言借用需要通过社交媒体API搜索台词引用频率
  优势：可量化、可统计
  局限：需要API权限或爬虫能力；国内平台API限制严格

离线手动模式（P2~P3级）：
  数据来源：分析者接触过的公开评论（非系统性采样）
  操作流程：
    Step 1 - 确定目标人物/维度：从E1中选择3-5个最核心的人物（如鹿鼎记的康熙、
             韦小宝、陈近南），或3-5个最核心的认知维度（如"明朝=腐朽"）
    Step 2 - 记忆采样：回忆你在豆瓣/知乎/B站/微博等平台见过的评论，对每个人物
             或维度，列出你记得的评论的"主流情感倾向"
    Step 3 - 等级标注：
             [压倒性多数]：我遇到的评论中80%以上是这种倾向
             [多数]：我遇到的评论中60-80%是这种倾向
             [分歧明显]：正反都有，无明显主流
             [样本不足]：我接触到的评论太少，无法判断
    Step 4 - 引用具体内容：如果记得具体评论内容，写下原文（或大意），标注
             "这是我记得的一条评论，可能在细节上有偏差"
    Step 5 - 批判意识检查：主动回忆是否有"抵抗式评论"（批评作品历史观的），
             比较其数量和质量与主流评论的关系
  局限：
    - 记忆偏差：你更容易记住与你的判断一致的评论
    - 幸存者偏差：你接触到的评论受你的社交圈和平台选择影响
    - E4-a和E4-b可部分完成（P2~P3级），E4-c不可完成（标记为缺口）
  注意事项：
    - 如果你自己也是受众，可以标注"作为受众，我的情感倾向是……"，但不能用
      自己的感受替代受众数据
    - "我没有看到过相反意见"不等于"没有相反意见"——标注为[样本不足]
      而非[压倒性一致]

大模型辅助模式（P2~P3级）：
  提示词结构：输入核心人物/维度清单，要求LLM基于训练语料中的公共讨论特征，
  输出对主流情感倾向的推断
  关键要求：
    - 要求LLM区分"训练语料中的常见评论"vs"自己的推断"
    - 要求LLM给出估计比例（如"约70%的公开评论对康熙持正面态度"）并标注
     依据
    - 要求LLM提供具体的评论类型示例（"常见评论有'千古一帝''好皇帝'等"）
  局限：
    - LLM训练数据有时间和平台偏差（可能过度代表某个平台的讨论）
    - LLM可能凭空编造"代表性评论"
    - 标注为[模型推断]，由分析者交叉检查
```

#### E4 证据产出格式

## 三、交叉验证规则

### 3.1 结论置信度等级

每条结论的置信度由**支持的独立组学维度数量**决定：

| 等级 | 条件 | 含义 | 使用限制 |
|------|------|------|---------|
| **S级** | 4个维度一致 | 结论可靠，可作为知识积累 | 可在无警告情况下引用 |
| **A级** | 3个维度一致 | 结论较可靠，一个维度缺失或未评估 | 引用时标注缺失维度 |
| **B级** | 2个维度一致 | 有两个独立来源支持 | 标记"待更多验证" |
| **C级** | 只有E1(文本) | 只有分析者判断，无独立验证 | 不纳入知识体系，仅作参考 |
| **D级** | 无独立证据 | 分析者猜测 | 不应出现在正式图谱中 |

### 3.2 冲突处理协议

当不同组学维度的结论冲突时：

```
检测到冲突 → 锁定冲突点 → 判断冲突类型 → 执行对应处理

冲突类型A：E1 ≠ E2（文本判断 vs 传播事实）
  处理：以E2为准，修正E1。
  理由：传播事实比文本解读更接近"模因在现实中的真实状态"。

冲突类型B：E1 ≠ E4（文本判断 vs 受众效果）
  处理：以E4为准，修正E1。
  理由：受众的实际反应比分析者的推测更有说服力。

冲突类型C：E2 ≠ E4（传播事实 vs 受众效果）
  处理：具体分析差异来源——可能由渠道筛选受众造成。
  不自动偏向任何一方，标记为"特性冲突，需更多数据"。

冲突类型D：E3 ≠ E4（调控因素 vs 受众效果）
  处理：同C，需具体分析。可能调控因素没有实际影响受众。
```

### 3.3 结论的层次结构

引入多组学后，模因图谱的结论不再是"分析者说了算"，而是分层输出：

```
第一层：观测数据（不可争议）
  E1 层0素材清单——这是事实，不是判断
  E2 传播数据——这是事实，不是判断
  E4 受众数据——这是事实，不是判断

第二层：分析判断（有E1-E2-E3-E4交叉验证支撑）
  E1 层1偏差标注
  E2 变异轨迹判定
  L2-L5的结构化分析
  这些判断的置信度由它们获得的一致组学维度数量决定

第三层：终极判定（综合结论）
  基于所有可用证据的最终判断
  必须附带"四组学一致性矩阵"和总体置信度等级
```

## 五、标准化的采集与产出

### 5.1 强制采集原则

E1-E4 四个组学维度的证据采集是每份模因图谱的**强制要求**。这不是"完整版 vs Lite版"的选择——只有一个版本，就是全量版。

### 5.2 无法采集时的处理

当某个维度的数据确实无法采集时：

| 情况 | 处理 |
|------|------|
| E2 无法采集（从未有公开传播数据） | 图谱最高置信度降为[C级] |
| E3 无法采集（社会调控数据不可得） | 图谱最高置信度降为[C级] |
| E4 无法采集（无受众数据） | 图谱最高置信度降为[C级] |
| E2+E3+E4 中多个无法采集 | 最高置信度降为[D级]，终极判定标记为"推测" |

条件限制可以接受，但必须诚实标注缺口并承担对应的置信度损失。

### 5.3 数据采集：三种模式

| 模式 | 适用场景 | E2可达等级 | E3可达等级 | E4可达等级 | 对结论置信度的影响 |
|------|---------|-----------|-----------|-----------|------------------|
| **联网模式** | 有网络、可访问公开数据库 | E2 P1~P2 | E3 P2 | E4 P1~P2 | 支撑 S~A 级 |
| **离线手动模式** | 无网络，仅靠分析者记忆+推理 | E2 P2~P3 | E3 P2 | E4 P2~P3 | 支撑 A~B 级 |
| **大模型辅助模式** | 有LLM但无联网搜索 | E2 P2~P3 | E3 P2~P3 | E4 P2~P3 | 支撑 A~B 级 |

各子维度的数据来源：

| 数据需求 | 联网模式来源 | 离线手动模式来源 | 大模型辅助模式来源 |
|---------|-------------|-----------------|------------------|
| E2-a 跨媒介复制 | 维基百科、豆瓣条目、影视数据库 | 分析者记忆+改编版对照 | LLM训练数据 |
| E2-b 变异检测 | 社交媒体API、B站、同人平台 | 记忆采样法 | LLM训练数据 |
| E2-c 表达量分析 | 分平台采样（豆瓣/知乎/微博/B站） | 受众分层推断（P4） | LLM对平台特征的推断 |
| E3-a 权力结构 | 出版史文献、文化产业研究 | 分析者独立分析（P2） | 分析者独立判断 |
| E3-b 渠道分析 | 传播学文献、平台研究 | 分析者独立分析（P2） | 分析者独立判断 |
| E3-c 竞争模因 | 同时代同类作品对照 | 分析者作品知识（P2） | LLM训练数据 |
| E4-a 情感表型 | 豆瓣短评、IMDb评论API | 记忆采样法（P3） | LLM训练数据 |
| E4-b 认知表型 | 知乎问答API | 记忆采样法（P3） | LLM训练数据 |
| E4-c 行为表型 | 仅追踪研究（待方法成熟） | 不可采集（缺口） | 不可采集（缺口） |

### 5.4 产出格式

每份模因图谱在完成 E1-E4 采集后，必须附带：

1. **采集模式声明**（每个E维度的采集模式——联网/离线手动/大模型辅助）
2. **每条结论的采集方法等级**（P1/P2/P3/P4）
3. **四组学一致性矩阵**（汇总表，每条结论附带P级标注）
4. **每层结论的证据等级**（S/A/B/C/D 标记）
5. **总体置信度**（终极判定的证据支撑情况）
6. **无法采集的维度**（诚实标注缺口）

## 六、设计问题（含方法论测试结论）

### 6.1 方法论测试结论

2026-05 对《鹿鼎记》进行了首次实案四组学交叉验证（详见《鹿鼎记》四组学交叉验证报告.md），核心发现：

1. **框架可行**——逻辑框架在实案中产出可用结论（S级 3/5，A级 2/5），五个染色体结论全部获得至少 3 维一致
2. **数据质量是瓶颈**——E2/E3/E4 的大部分数据来自分析者常识或定性观察，而非系统采集的定量数据。当前证据标准缺乏对"采集方法质量"的标记
3. **"常识"数据可能不独立**——分析者的"常识"可能正是通过 E1 文本分析构建的认知框架，再用它来验证 E1 会造成循环论证
4. **工具缺失是核心障碍**——E2（社交媒体API）、E4（情感分析工具）的系统采集需要工具支持

### 6.2 改进：数据采集方法等级

基于测试结论，E2/E3/E4 的每条结论必须附带**数据采集方法等级**标记，与结论置信度等级（S/A/B/C/D）并行使用：

| 采集方法等级 | 含义 | 独立性风险 | 适用场景 |
|-------------|------|-----------|---------|
| **P1**（API系统采集） | 通过 API 或爬虫获取的定量数据 | 低——数据独立于分析者 | 搜索引擎趋势、豆瓣/B站 API |
| **P2**（手动系统采样） | 人工抽样 + 编码框架的手动分析 | 中——编码框架依赖分析者但数据来自外部 | 评论内容分析、改编版本逐版对照 |
| **P3**（分析者观察） | 基于分析者印象和常识的定性判断 | 高——可能混入 E1 的认知污染 | 功能转化观察、意义漂变印象 |
| **P4**（理论推断） | 无数据支撑的逻辑推断 | 极高——实质上是 E1 的延伸 | 代际差异推测、平台画像推断 |

**使用规则**：
- P3 和 P4 的数据**不能作为独立验证**——只能作为"线索"和补充
- S 级结论至少需要 E2/E3/E4 中有两项达到 P2 或以上
- P4 数据必须标注"推断"，不得用于一致性矩阵的"一致"判定

### 6.3 改进：E4-c 行为表型的地位

E4-c（长期社会行为影响）在当前技术条件下不可系统性采集。因此：

- E4-c 不作为 S 级判定的必要条件
- 当 E4-c 无法采集时，E4 维度的判定权重降低，E4-a + E4-b 的 P2 级数据可视为"E4 维度的有效支持"
- E4-c 标记为"长期研究课题"，但在作品级图谱中必须声明其缺失

### 6.4 改进：独立性校验机制

鉴于"常识级数据可能不独立于 E1"的问题，增加以下校验规则：

- **交叉检查**：E2/E3/E4 的 P3 级数据，分析者必须在采集时声明"这个数据是独立于 E1 分析获得的吗？"
  - 如果是：标注[独立获取]
  - 如果否（如"我在读作品时就已知道这个传播事实"）：标注[非独立获取]——不能作为独立验证维度
- P1/P2 级数据默认视为独立获取
- P4 级数据默认视为非独立获取

### 6.5 未解决的设计问题

1. **E4-c 受众行为表型的采集方法**：当前最深的"长期社会行为影响"无法用公开数据观测，需要追踪研究。这是已知局限——不在本版本解决，标记为"待方法成熟"。

2. **四组学工作量的可行性**：一份图谱的 E1 文本分析约需 10-20 小时（L1 全量测序）。E2+E3+E4 的采集如果按 P2 标准（手动系统采样），预计增加 20-40 小时。这是否在可接受范围内？如果不可接受，工具化是唯一的出路。

3. **E3 调控组学的客观性**：E2（传播数据）和 E4（受众数据）相对客观。E3（社会结构分析）本质上依赖分析者的判断——即使上升到 P1 级，社会结构的"数据"也需要分析者选择什么数据是相关的。E3 的独立性是相对的，不可与 E2 和 E4 等同。

4. **效率与深度的平衡**：如果每部作品都需要全量四组学分析（P2 级标准），那分析的广度（跨作品比较）必然严重受限。是否应该建立一个"轻量版"的快速扫描标准——只做 P3 级观察，但不纳入知识体系？这是对"不要 Lite"原则的补充而非否定：轻量版的作用是"筛选值得深度分析的作品"，而不是"替代全量分析"。
